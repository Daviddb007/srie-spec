# SRIE Constitution

## Preamble

SRIE (Software Resource Intelligence Engine) is a meta-system for project governance,
engineering intelligence, and lifecycle management. This repository contains the
frozen specification — the "Constitution" — versioned and stable.

## Architectural Evidence Rule

> No document in this repository may be modified solely because it "seems better."
> It may only be modified when an implementation or real-world use case demonstrates
> that the current design is insufficient.

## Change Process

1. A real implementation must encounter a limitation not addressed by the spec.
2. Evidence must be documented showing why the spec is insufficient.
3. A PR modifies the relevant document with rationale and evidence.
4. The PR is reviewed and merged only if the evidence is accepted.

## Versioning

This repository follows semantic versioning for the specification.
- v1.0: Initial frozen specification (Sprint 0)
